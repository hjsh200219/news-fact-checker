# 성숙도 프레임워크 — news-fact-checker

L1–L5 성숙도를 4개 차원으로 평가한다. 각 차원은 독립 등급, 종합은 가중평균.
채점은 [principles.md](principles.md)의 회의적 평가 원칙을 따른다("증명될 때까지 미흡").

## 4개 차원 + 가중치

| 차원 | 가중치 | 무엇을 보는가 |
|------|-------:|---------------|
| D1. 문서 / 진입 | 20% | 진입 문서가 map-not-handbook인가, SSOT 링크 유효한가 |
| D2. 아키텍처 강제 | 30% | 불변식이 코드/스크립트로 강제되는가(문서 규칙만이면 감점) |
| D3. 테스트 / 검증 | 30% | 계약 테스트가 실패 분기를 네트워크 없이 잠그는가, CI 게이트 있는가 |
| D4. 관측성 / 재현성 | 20% | 기계 상태 계약 + provenance로 결과를 사후 감사할 수 있는가 |

## L1–L5 등급 정의

| L | 이름 | 일반 정의 |
|---|------|-----------|
| L1 | Ad-hoc | 규칙이 사람 머릿속/프롬프트에만. 재현·검증 불가 |
| L2 | Documented | 규칙이 문서화됨. 강제·테스트는 부분/부재 |
| L3 | Enforced | 핵심 불변식이 코드로 강제 + 일부 계약 테스트 |
| L4 | Gated | 전 핵심 분기 계약 테스트 + CI 게이트 + 기계 상태 계약 |
| L5 | Self-improving | 가정 재검증 루프·자동 성숙도 추적·drift 알림까지 상시 운영 |

## 차원별 L1–L5 앵커

### D1 문서 / 진입
- L1 진입 문서 없음 · L2 있으나 handbook화/링크 깨짐 · L3 맵+일부 링크 · L4 ~100줄 맵 + Invariants + Health Stack + 유효 링크 · L5 문서-코드 일치 자동 검증(verify-docs).

### D2 아키텍처 강제
- L1 프롬프트뿐 · L2 문서 규칙 · L3 판정 게이트 등 일부 코드화 · L4 판정·SSRF·공급망·상태 전부 스크립트 강제 · L5 강제 가정의 주기적 재검증(하네스 단순화 루프).

### D3 테스트 / 검증
- L1 없음/네트워크 의존 · L2 산발적 · L3 계약 테스트 일부·CI 없음 · L4 전 분기 네트워크-free 계약 테스트 + selftest + Linux/macOS CI + 로컬 미러 · L5 mutation/커버리지 게이트·회귀 자동 추적.

### D4 관측성 / 재현성
- L1 재현 불가 · L2 비구조 로그 · L3 부분 상태 · L4 `NFC_STATUS` 기계 계약 + `.nfc-provenance.json` 리포트 흐름 · L5 실행 간 diff 자동 감사(엔진 변화 vs 콘텐츠 변화).

## 현재 등급 (정직 평가 — 기준 커밋 6d09fac + 미커밋 트리)

| 차원 | 등급 | 근거 |
|------|:----:|------|
| D1 문서 / 진입 | **L2 → L4** | 감사 시점 repo-level 진입 문서 부재(L2). 이 Phase에서 AGENTS.md/ARCHITECTURE.md/docs 신설로 L4 도달. 커밋 시 확정. |
| D2 아키텍처 강제 | **L4** | `verdict_gate`(판정)·`url_policy.py`(SSRF)·`resolve_engine.sh`(commit-pin+atomic)·`parse_engine_status.py`(drift) 모두 코드 강제. 가정 재검증 루프 미상설이라 L5 아님. |
| D3 테스트 / 검증 | **L3 (→L4 조건부)** | 네트워크-free 계약 테스트 + selftest + Linux/macOS CI 존재. **단 TD-1: 테스트/CI가 아직 untracked** — 원격에서 실제 게이트되기 전까지 L4 확정 보류. |
| D4 관측성 / 재현성 | **L4** | `NFC_STATUS` 스키마 + provenance가 상태·리포트로 흐름. 실행 간 자동 diff 감사는 없어 L5 아님. |

### 종합

가중평균 (커밋 후 D1=4, D3=4 가정): `0.2·4 + 0.3·4 + 0.3·4 + 0.2·4 = 4.0` → **L4 (Gated)**.

현 미커밋 상태 기준(D1=2, D3=3): `0.2·2 + 0.3·4 + 0.3·3 + 0.2·4 = 3.3` → **L3 상단**.

**결론: 강제·관측은 이미 L4급. 병목은 D1(진입 문서, 이 Phase에서 해소) + D3의 TD-1(미커밋 산출물).
TD-1 커밋 후 재평가하면 종합 L4가 확정된다.** L5 도달에는 하네스 단순화 재검증 루프(D2) +
실행 간 diff 감사(D4) + 문서-코드 자동 검증(D1)이 필요.
