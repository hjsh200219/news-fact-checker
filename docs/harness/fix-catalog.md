# Fix 카탈로그 — news-fact-checker

원칙(차원) × 점수 구간별 개선 액션 카탈로그. `/sh:harness-gc`가 낮은 점수 차원을 만나면 여기서
구체 액션을 뽑아 Sprint Contract(기대 효과 사전 합의)로 실행한다.

## 사용법 (템플릿)

각 항목:
- **차원 / 점수대:** 어떤 차원이 몇 점일 때
- **증상:** 관측 가능한 신호
- **액션:** 구체적 수정
- **기대 효과 (Sprint Contract):** 어떤 게이트/점수가 어떻게 바뀌는가
- **검증:** 통과를 어떻게 확인하는가

```
### <제목>
- 차원 / 점수대:
- 증상:
- 액션:
- 기대 효과:
- 검증:
```

---

## D1 문서 / 진입

### 진입 문서 부재 (L1–L2)
- **차원 / 점수대:** D1 · 0–2
- **증상:** repo-level `AGENTS.md`/`ARCHITECTURE.md` 없음. 기여자가 SKILL.md만 보고 아키텍처 추론.
- **액션:** `AGENTS.md`(맵, ~100줄) + `ARCHITECTURE.md`(레이어/의존성/경계) 신설, docs/ SSOT로 링크.
- **기대 효과:** D1 → L4. 온보딩 진입점 확보.
- **검증:** 링크 유효성, 줄 수 ≤~120, Invariants + Health Stack 존재.

### handbook화 (L2–L3)
- **차원 / 점수대:** D1 · 2–3
- **증상:** AGENTS.md에 상세 표/규칙 인라인, 100줄 초과.
- **액션:** 상세를 docs/ 하위로 이동하고 링크로 대체.
- **기대 효과:** D1 → L4. 진입 문서 유지비 감소.
- **검증:** AGENTS.md 줄 수, 중복 섹션 제거 확인.

## D2 아키텍처 강제

### 규칙이 문서에만 있음 (L2)
- **차원 / 점수대:** D2 · 1–3
- **증상:** "이렇게 하라"가 프롬프트/문서에만 있고 위반을 잡는 코드/테스트 없음.
- **액션:** 불변식을 순수함수 + 픽스처로 승격(예: 판정 게이트 → `verdict_gate`, 목적지 → `url_policy.py`).
- **기대 효과:** D2 → L4. 프롬프트 우회 불가.
- **검증:** 위반 입력이 exit≠0 또는 `검증불가`로 강제되는 테스트.

### 강제 가정 재검증 부재 (L4→L5)
- **차원 / 점수대:** D2 · 4
- **증상:** 스크립트가 인코딩한 "모델이 못하는 것" 가정이 오래되어 불필요하게 복잡.
- **액션:** 하네스 단순화 원칙 적용 — 각 스크립트의 가정을 현재 모델로 재검증, 불필요분 제거.
- **기대 효과:** D2 → L5. 복잡도 감소, 유지비↓.
- **검증:** 제거 후에도 `--selftest`/계약 테스트 전부 통과.

## D3 테스트 / 검증

### 미커밋 산출물 (현재 TD-1)
- **차원 / 점수대:** D3 · 3
- **증상:** 테스트/CI가 로컬엔 있으나 git untracked → 원격에서 게이트 안 됨.
- **액션:** `scripts/url_policy.py`·`parse_engine_status.py`·`tests/`·`.github/`·`docs/`를 커밋(TD-1).
- **기대 효과:** D3 → L4 확정. CI가 실제로 계약을 강제.
- **검증:** `git ls-files`에 존재 + `bash plugins/news-fact-checker/tests/run.sh` 통과.

### 네트워크 의존 테스트
- **차원 / 점수대:** D3 · 2–3
- **증상:** 테스트가 실 네트워크/실 engine을 요구해 CI에서 불안정.
- **액션:** `fake_engine` fixture를 확장해 env 시나리오로 분기 구동.
- **기대 효과:** D3 → L4. 결정론·재현성.
- **검증:** 네트워크 차단 환경에서 전 테스트 통과.

## D4 관측성 / 재현성

### provenance 미흐름 (L2–L3)
- **차원 / 점수대:** D4 · 1–3
- **증상:** 리포트에 engine version/commit이 없어 결과 차이 원인 추적 불가.
- **액션:** `.nfc-provenance.json` → `NFC_STATUS.engine_version/commit` → 리포트 헤더로 흐르게 유지.
- **기대 효과:** D4 → L4. 사후 감사 가능.
- **검증:** 리포트에 version/commit 필드 렌더 확인.

### 실행 간 diff 감사 부재 (L4→L5)
- **차원 / 점수대:** D4 · 4
- **증상:** 같은 URL 재검증 시 결과 차이가 엔진 변화인지 콘텐츠 변화인지 구분 불가.
- **액션:** provenance + 판정 스냅샷 비교 도구 도입.
- **기대 효과:** D4 → L5.
- **검증:** 엔진 commit 고정 시 판정 안정성 재현.
