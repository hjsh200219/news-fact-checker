# 출처 독립성 붕괴 (independence.py)

## 왜
한국 뉴스는 **통신사 신디케이션**이 지배적이다. 연합뉴스/뉴스1/뉴시스 한 기사가 리드·제목을
바꿔 20개 매체에 재출고된다. 원시 "출처 수"로 세면 **거짓 독립성**이 생긴다 — 사실상 1개 출처를
20개로 착각. 그래서 "독립 출처 ≥2"를 강제하기 **전에** 신디케이션 클러스터를 1개 유효 출처로 붕괴한다.

## 사용
수집한 확증 출처를 다음 형식의 JSON 리스트로 만들어 stdin으로 넣는다:
```bash
echo '[{"url":"...","title":"...","body":"...","byline":"...","dateline":"..."}, ...]' \
  | python3 "${SKILL_DIR}/../../scripts/independence.py"
```
출력: `{"clusters": [[idx,...], ...], "effective_count": N}`
- `clusters`: 같은 유효 출처로 묶인 입력 인덱스 그룹.
- `effective_count`: **유효(독립) 출처 수**. 이 값이 ≥2일 때만 주장을 `사실`/`거짓`으로 확정할 수 있다.

## 붕괴 규칙 (wire-token dominant)
두 출처를 링크(같은 클러스터)하는 조건 — 하나라도 참이면 링크:
1. **공유 wire 토큰 있음 AND 문자-trigram 유사도 ≥ 0.35** — 신디케이션(리드 rewrite로 유사도가
   순수 중복 임계값보다 낮아지므로, wire 토큰이 있으면 문턱을 낮춘다). **이것이 지배적 신호.**
2. **byline 일치 AND dateline 일치** — 같은 기자·같은 데이트라인.
3. **문자-trigram 유사도 ≥ 0.85** — 순수 근접 중복(그대로 복붙).

연결 요소(union-find)가 클러스터. `effective_count` = 클러스터 수.

## 신호 설명
- **wire 토큰**: `연합뉴스`, `뉴스1`, `뉴시스`, `(서울=`, `제공`, `AP`, `Reuters`, `로이터`, `AFP`,
  `블룸버그` 등(스크립트 `WIRE_TOKENS`). 기사 body/byline/dateline/title에서 탐지.
- **문자 n-gram(k=3)**: 공백 제거 정규화 텍스트의 문자 트라이그램 Jaccard. 한국어 조사 결합·어순
  변화에 강함(단어 n-gram은 rewrite에 취약해 부적합).
- **주제 게이팅**: 같은 wire 토큰이라도 주제가 다르면(예: "물가 대책" vs "태풍 북상") 유사도가 낮아
  붕괴하지 않는다 — 같은 통신사의 서로 다른 기사를 합치지 않는다.

## stance (별도, 스크립트 밖)
붕괴와 별개로, 각 출처가 주장을 **지지/반박/무관(supports/refutes/unrelated)** 하는지 분류한다.
`unrelated`(키워드만 걸린 무관 출처)는 카운트에서 제외 — 근거 인플레이션 방지.

## 임계값 튜닝
`TAU`(0.85), `TAU_WIRE`(0.35), `CHAR_K`(3)은 스크립트 상단 상수. `--selftest`가 3개 픽스처
(신디케이션 붕괴 / 독립 2건 / 같은-wire-다른-주제 비붕괴)로 회귀를 지킨다.
